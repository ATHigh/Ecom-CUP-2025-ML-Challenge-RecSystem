# E-CUP-2025-ML-Challenge-RecSystem-

Проблематика
Ассортимент fashion‑товаров на Ozon превышает 150 млн позиций, поэтому выбор превращается в многоминутный скроллинг и теряется конверсия. Персональные рекомендации снимают информационную перегрузку, повышают удовлетворенность клиентов и прямо влияют на выручку площадки.

Полное описание задачи
Создайте масштабируемую user-to-item рекомендательную систему для категории Apparel (одежда, обувь, аксессуары), которая должна:

Интегрировать разнородные источники данных:
1,6 млрд взаимодействий пользователей (просмотры, добавления в корзину, избранное, покупки) за 6 месяцев;
19 млн заказов с финальными статусами (доставлен, отменён, в процессе);
6,4 млн товаров с каталожными атрибутами (текстовые, числовые, категориальные признаки);
Готовые CLIP-эмбеддинги изображений товаров (512 измерений), дообученные под fashion-сегмент;
Иерархию категорий товаров (7 тыс. категорий).
Сформировать персональный топ-100 предсказаний для каждого пользователя тестовой выборки, где предсказываются товары, которые будут куплены с финальным статусом «доставлен».
Построить полный end-to-end pipeline (подготовка данных → feature engineering → обучение → инференс).
Реализовать обработку больших данных (~38 ГБ в формате parquet).
Обеспечить временную валидацию с учётом хронологии взаимодействий.
Допускать использование GPT-подобных моделей только как вспомогательных компонентов, а не ядра решения.
Предоставить рабочий репозиторий (GitHub, GitLab, Bitbucket) с README и скриптами для воспроизводимости.
Бизнес требования
Генерация топ-100 предсказаний для каждого пользователя тестовой выборки с выгрузкой CSV submission в формате: 
user_id,item_id_1 item_id_2 ... item_id_100
Эффективная обработка больших parquet-файлов;
Соблюдение временной последовательности данных;
Репозиторий с pyproject.toml для развёртывания окружения;
Отсутствие внешних API-вызовов в финальном запуске;
GPT‑like модели допускаются только как вспомогательный компонент, не как ядро решения.
Технические требования и ограничения
Язык: Python ≥ 3.10;
Формат данных: Parquet (требуется оптимизация чтения и обработки);
Разрешены любые open-source библиотеки из PyPI/Conda (pandas, polars, pyspark, pytorch и др.);
Запрещены внешние API-запросы при финальном запуске;
Желателен pyproject.toml; окружение поднимается с Poetry.
